# 中文文笔优化器

> 基于优质中文范例优化文章文笔
>
> **⚠️ 核心原则：只优化文笔和表达方式，绝不改变原有框架和结构**

---

## 一、核心原则（重要！）

### ✅ 做什么

| 优化项 | 说明 |
|--------|------|
| 句式调整 | 把生硬的句子改自然 |
| 口语化 | 去翻译腔、去书面语 |
| 情绪增强 | 加情绪词、感叹、反问 |
| 画面感 | 把抽象变具体 |
| 节奏感 | 长短句交替 |
| 金句打磨 | 提炼可传播的句子 |

### ❌ 不做什么

| 禁止项 | 说明 |
|--------|------|
| 不改结构 | 保留原有段落划分 |
| 不改框架 | 保留 PASTOR/AIDA 等框架 |
| 不改逻辑 | 保留原有论证顺序 |
| 不改观点 | 保留原有核心观点 |
| 不改篇幅 | 保持原有字数范围 |
| 不删内容 | 只优化表达，不删减内容 |

---

## 二、使用方法

### 标准流程

```
步骤1：用 Dankoe Writer 生成初稿（结构+框架已定）
```

## 四、文笔优化技巧

### 4.1 情绪煽动（咪蒙风格）

**技巧：制造对立**

| 平淡 | 有力 |
|------|------|
| 有些人不努力 | 你不是不努力，你是假装在努力 |
| 这样做不对 | 这样做的人，都后悔了 |
| 很多人不知道 | 只有1%的人知道这个真相 |

**技巧：情绪放大**

| 平淡 | 有力 |
|------|------|
| 这让人很生气 | 我气得把手机都摔了 |
| 这件事很遗憾 | 这件事让我难过了整整三年 |
| 她很开心 | 她笑得像朵花，眼角都笑出了皱纹 |

### 4.2 节奏控制

**短句轰炸**
```
平淡：我觉得这件事情真的很重要，大家应该认真对待。
有力：这件事很重要。真的很重要。你必须认真对待。
```

**长短交替**
```
长句解释，短句强调。

当你开始用系统的思维看待问题，你会发现所有看似孤立的事件其实都是相互关联的。
改变一个环节就会产生连锁反应。这就是系统。
```

### 4.3 口语化表达

| 书面语 | 口语化 |
|--------|--------|
| 因此 | 所以嘛 |
| 然而 | 但是呢 |
| 综上所述 | 说白了就是 |
| 由此可见 | 你看 |
| 值得注意的是 | 有个事儿你得知道 |

### 4.4 故事驱动

**公式：具体场景 + 细节 + 情绪**

```
平淡：她很焦虑。

有画面：凌晨三点，她盯着天花板，脑子里像有一百个标签在转。

有情绪：凌晨三点，她盯着天花板。第二天要交方案，孩子还发烧，老公出差了。
她想起妈妈说过的话："女人啊，就是命苦。"眼泪就下来了。
```

### 4.5 金句公式

**对比型**
```
大多数人X，高手却在Y
例：大多数人追求更多，高手却在做减法
```

**否定型**
```
X？错。Y。
例：努力就能成功？错。方向比努力重要100倍。
```

**重定义型**
```
X不是Y，而是Z
例：睡眠不是任务，是礼物
```

## 五、优化工作流

### 输入文章后，AI 执行：

```
第1步：识别结构（只读不改）
- 标记段落划分
- 标记使用的框架
- 标记论证逻辑

第2步：逐句优化文笔
- 是否有翻译腔？→ 改口语
- 是否太平淡？→ 加情绪
- 是否太抽象？→ 加画面
- 是否缺少节奏？→ 调长短句

第3步：保留原有元素
- 保留所有段落
- 保留所有观点
- 保留所有框架位置
- 保留原篇幅

第4步：输出优化版
```

## 八、与 Dankoe Writer 的分工

```
Dankoe Writer 负责：
├─ 结构设计
├─ 框架选择（PASTOR/AIDA等）
├─ 逻辑安排
├─ 观点输出
└─ 篇幅控制

本模块负责：
├─ 句式优化
├─ 口语化调整
├─ 情绪增强
├─ 画面感添加
└─ 金句打磨

两者独立，各司其职，互不干涉
```

### 典型使用流程

```
用户：用 Dankoe 风格写一篇关于"拖延症"的文章

AI：[Dankoe Writer 生成]
    结构：钩子 → 问题 → 原因 → 方案 → 行动
    框架：PASTOR
    [初稿完成]

用户：优化文笔

AI：[本模块优化]
    保持结构不变 ✓
    保持框架不变 ✓
    只优化句式和表达 ✓
    [优化版完成]
```
